# 舆情监测日报处理 Skill

## 任务目标
- 本 Skill 用于：处理舆情监测日报 Word 文档，提取关键信息并生成标准化文字描述
- 能力包含：文档内容读取、结构化信息提取、文字描述生成
- 触发条件：用户提供舆情监测日报文档（.docx格式）并需要生成文字描述

## 前置准备
- 依赖说明：scripts 脚本所需的依赖包及版本
  ```
  python-docx==1.1.0
  ```

## 操作步骤

### 标准流程

**步骤 1：读取输入文件**
- 读取监测日报文档：调用 `scripts/read_docx.py <文档路径>` 提取文档内容
- 读取高频正面信息链接文件：直接读取 .txt 文件，每行一个链接

**步骤 2：提取关键信息**
智能体根据 [references/format-guide.md](references/format-guide.md) 中的格式规范，从文本中提取以下关键信息：
- 监测时间段
- 传播概述（声量统计、正负面占比、主要话题）
- 舆情数据（全网声量、正面/负面声量、评论监控）
- 高频正面信息标题（从日报文档提取）
- 高管高频正面信息

**步骤 3：匹配标题和链接**
- 将日报文档中的高频正面信息标题与链接文件中的URL按顺序匹配
- 第1个标题对应第1个链接，第2个标题对应第2个链接，以此类推

**步骤 4：判断高管高频正面信息**
- 从高频正面信息标题中筛选包含"何刚"或"余承东"的新闻
- 如果标题包含这两个关键词，则该条目属于高管高频正面信息
- 如果没有匹配的标题，则显示"无"

**步骤 5：生成并保存输出文件**
生成三个文档：

**文件1：舆情监测日报_[日期].txt**（完整文字描述）
```
【MM月DD日机关重点舆情监测日报】
MM月DD日HH:MM-MM月DD日HH:MM

传播概述：全网关于Y总、H总相关报道[总声量]篇次，正面/中性[正面声量]篇占比[正面占比]%；负面[负面声量]篇占[负面占比]%。正面新闻主要聚焦："[正面话题1]"、"[正面话题2]"、"[正面话题3]"、"[正面话题4]"、"[正面话题5]"等话题。负面新闻主要聚焦："[负面话题1]"、"[负面话题2]"、"[负面话题3]"等相关敏感内容。

【舆情数据】
1、全网声量：全网关于Y总、H总相关报道为[总声量]频次。
2、正面声量：正面/中性信息[正面声量]频次，占[正面占比]%。负面声量：负面信息[负面声量]频次，占[负面占比]%。
3、评论监控：监测时间段内，用户评论共[评论总数]条，正面/中性占比[正面评论占比]%（[正面评论数]条），负面占比[负面评论占比]%（[负面评论数]条），[是否有集中吐槽内容]。

【高频正面信息】
1⃣[标题1]
[链接1]
2⃣[标题2]
[链接2]
3⃣[标题3]
[链接3]
4⃣[标题4]
[链接4]
5⃣[标题5]
[链接5]

【高管高频正面信息】
[从高频正面信息中筛选包含"何刚"或"余承东"的标题，如无则显示"无"]
```

**文件2：[日期]日报概述.docx**
```
【MM月DD日机关重点舆情监测日报】
MM月DD日HH:MM-MM月DD日HH:MM

传播概述：全网关于Y总、H总相关报道[总声量]篇次，正面/中性[正面声量]篇占比[正面占比]%；负面[负面声量]篇占[负面占比]%。正面新闻主要聚焦："[正面话题1]"、"[正面话题2]"、"[正面话题3]"、"[正面话题4]"、"[正面话题5]"等话题。负面新闻主要聚焦："[负面话题1]"、"[负面话题2]"、"[负面话题3]"等相关敏感内容。

【舆情数据】
1、全网声量：全网关于Y总、H总相关报道为[总声量]频次。
2、正面声量：正面/中性信息[正面声量]频次，占[正面占比]%。负面声量：负面信息[负面声量]频次，占[负面占比]%。
3、评论监控：监测时间段内，用户评论共[评论总数]条，正面/中性占比[正面评论占比]%（[正面评论数]条），负面占比[负面评论占比]%（[负面评论数]条），[是否有集中吐槽内容]。
```

**文件3：[日期]高频正面信息.docx**
```
【高频正面信息】
1⃣[标题1]
[链接1]
2⃣[标题2]
[链接2]
3⃣[标题3]
[链接3]
4⃣[标题4]
[链接4]
5⃣[标题5]
[链接5]
```

**重要规则**：
- 标题格式固定为：【MM月DD日机关重点舆情监测日报】
- 主体统一使用"Y总、H总"
- 传播概述中正面话题列出5条，负面话题列出3条
- 高频正面信息固定列出5条，每条包含标题和链接
- 高管高频正面信息：从高频正面信息中筛选包含"何刚"或"余承东"的标题
- 格式严格遵循模板，不可改动

## 资源索引
- 必要脚本：见 [scripts/read_docx.py](scripts/read_docx.py)（用途：读取 .docx 文档并提取纯文本内容；参数：文档路径）
- 格式规范：见 [references/format-guide.md](references/format-guide.md)（何时读取：理解日报文档结构和字段定义时）

## 注意事项
- 文档格式必须符合舆情监测日报的标准格式
- 提取信息时保持数据准确性，尤其是数字和百分比
- 生成文字描述时保持原文的专业表述风格
- 如果某些字段缺失，在输出中标注"未提供"

## 使用示例
用户提供舆情监测日报文档后，智能体将：
1. 使用脚本读取文档内容
2. 识别并提取所有关键信息
3. 按照模板格式生成标准化文字描述
4. 输出可直接使用的文字内容
